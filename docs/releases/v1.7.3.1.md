### 中文

- 更新版本到 `1.7.3.1`，整合录制、窗口交互、预览和发布说明格式优化。
- 新增直播间局部设置入口，支持为单个直播间配置监控、录制、录制格式、清晰度、分段、轮询周期、轮询时段、保存目录和命名规则。
- 修复直播间局部设置实际写入全局配置的问题，局部录制格式、清晰度、分段、轮询、保存目录和命名规则会随房间保存并在解析与录制时生效。
- 优化添加直播间流程，可在添加时直接选择是否跟随全局设置，以及是否启用监控和录制。
- 扩展录屏列表操作，新增视频分割、连续片段合并、合并条件校验和风险提示。
- 补全录屏列表分割操作的确认入口和执行逻辑，并限制缩略图与媒体探测并发，避免视频较多时同时拉起过多 FFmpeg/FFprobe 进程。
- 加强连续片段合并校验，阻止不同视频格式的片段直接合并，降低 `-c copy` 失败或输出异常的风险。
- 优化录屏列表与设置窗口的浮层交互，弹窗会居中显示并随窗口尺寸变化重新适配。
- 修复顶部运行按钮悬停样式，统一主界面交互状态。
- 增强录制稳定性：解析失败时保留正在录制的可用流地址，避免短暂解析异常打断录制状态。
- 录制结束后自动清理没有对应视频文件的孤立元数据，减少无效 `.mplr.json` 文件残留。
- 优化外部直播流解析器的临时配置文件处理，避免并发解析时共享配置互相覆盖，并自动清理过期配置。
- 优化配置变更日志读取，减少配置文件短暂占用时导致的读取失败。
- 加速预览启动：启动后后台预热解析器，缓存 Python、解析脚本、`ffplay` 和 `ffprobe` 路径，减少重复进程探测。
- 优化预览解析逻辑：已有可用流地址时优先直接打开，多个候选流并发快速探测，降低预览前等待时间。
- 修复预览窗口刚打开后手动拖动会被自动复位的问题。
- 调整 GitHub Release 自动生成正文格式，移除重复版本信息和正文贡献者段，降低标题层级以避免多余下划线。

### English

- Bumped the version to `1.7.3.1` and consolidated recording, window interaction, preview, and release-note formatting updates.
- Added a per-room local settings entry for monitor, recording, record format, stream quality, segmentation, polling interval, polling schedule, save folder, and filename rules.
- Fixed local room settings being written to global configuration; per-room format, quality, segmentation, polling, save folder, and filename rules are now stored with the room and applied during resolving and recording.
- Improved the add-room flow so monitor/record options and global-setting inheritance can be selected while adding a room.
- Expanded the screen-record list with video splitting, continuous segment merging, merge validation, and warning messages.
- Completed the split-video confirmation and execution path, and throttled thumbnail/probe enrichment to avoid spawning too many FFmpeg/FFprobe processes at once.
- Strengthened merge validation by rejecting mixed video formats before running stream-copy concatenation.
- Improved floating-panel behavior in the screen-record list and settings windows so panels stay centered and responsive to window resizing.
- Fixed the hover style of the top run button for more consistent main-window interaction states.
- Improved recording resilience by preserving usable stream URLs while a room is already recording if a later resolver check temporarily fails.
- Removed orphan recording metadata when no matching video file is produced.
- Improved external stream resolver config handling so concurrent resolver processes use isolated temp configs and stale configs are cleaned up.
- Hardened config-change logging against transient config-file locks.
- Sped up preview startup by warming the resolver in the background and caching Python, resolver script, `ffplay`, and `ffprobe` lookups.
- Improved preview resolution flow by opening cached stream URLs first and probing multiple candidate streams in parallel with shorter timeouts.
- Fixed preview windows snapping back after being manually moved right after opening.
- Refined generated GitHub Release notes by removing repeated version info and duplicate contributors content, and by using softer heading levels to avoid extra underlines.
