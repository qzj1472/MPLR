### 中文

### 更新范围

- 本日志汇总自 `v1.7.4.3` 之后至 `v1.7.5` 的累计更新，包含录制与转码修复、预览延迟优化、Velopack 自动更新、正式版/Beta 发布通道和设置页交互整理。

### 录制、转码与媒体处理

- 优化添加直播间与剪贴板导入流程，减少快速添加或复制链接时的界面卡顿和状态错乱。
- 修复转码时媒体流映射不完整的问题，保留输入文件中需要继承的音视频流，降低转码后音轨或媒体流丢失风险。
- 修复双音轨录制内容在转码后只保留单音轨的问题，提升多音轨直播录制的输出完整性。

### 预览与播放体验

- 优化内嵌预览播放器的低延迟参数，降低直播预览缓冲和卡顿概率。
- 调整预览启动后的播放链路，减少打开预览时对主界面加载和操作响应的影响。
- 继续保留内嵌播放方案，避免外部 `ffplay` 窗口散落在任务管理器中，预览行为更贴近软件内部播放器。

### 自动更新与发布通道

- 集成 Velopack 自动更新能力，关于面板可显示当前安装包通道并支持手动检查更新。
- 新增稳定版与 Beta 更新通道：稳定版使用 `win`，Beta 使用 `win-beta`，用户可在设置中切换通道。
- Beta 版本或 Beta 通道会强制开启运行日志和强制自动更新，方便测试版本定位问题并保持测试用户处于最新构建。
- 切换更新通道时会弹窗说明影响，确认后立即保存配置并检查目标通道更新；从 Beta 切回稳定版时会通过自动安装稳定版完成通道切换。
- 统一应用版本、包版本和构建通道元数据，正式版显示 `v1.7.5`，Beta 构建显示带 Beta 标识的包版本。

### 设置、日志与开发者入口

- 调整设置页卡片顺序，将语言、主题、更新通道、运行日志、托盘、桌面快捷方式、配置文件和高级工具按使用频率重新组织。
- 运行日志卡片会根据当前更新策略显示是否可关闭；Beta 强制日志时会禁用关闭开关并给出明确提示。
- 新增开发者模式持久化状态，并将开发者工具入口隐藏到低干扰位置，避免普通用户界面过于显眼。
- 关于窗口补充构建通道显示，便于确认当前安装包来自稳定版还是 Beta 通道。

### 构建、安装包与 GitHub Release

- 工作流支持正式版和 Beta 两套发布输入，并在构建、测试、发布、Velopack 打包和上传阶段传递一致的版本与通道参数。
- 正式版默认发布到 `v1.7.5`，Beta 默认发布到 `v1.7.5-beta`，Beta 包版本使用 `1.7.5-beta.{run_number}` 以支持连续升级。
- Beta 发布不再依赖手写 `docs/releases/*.md`，工作流会自动生成简短说明和提交列表。
- 发布 Beta 前会清理旧 Beta release/tag，再重新创建当前 Beta 预发布，GitHub 上默认只保留一个 Beta 发布槽。
- 重新整理发布产物，保留自定义安装器 EXE、Portable ZIP、Velopack 完整包和更新清单，便于安装、免安装使用和软件内自动更新。

### English

### Scope

- This changelog summarizes updates from `v1.7.4.3` through `v1.7.5`, covering recording and transcoding fixes, preview latency improvements, Velopack automatic updates, stable/Beta release channels, and settings interaction cleanup.

### Recording, Transcoding And Media Processing

- Improved add-room and clipboard import flows to reduce UI stalls and state confusion during quick link entry.
- Fixed incomplete media stream mapping during transcoding so expected input audio/video streams are preserved more reliably.
- Fixed dual-audio recordings losing one audio track after transcoding, improving output integrity for multi-audio live recordings.

### Preview And Playback

- Tuned the embedded preview player for lower live playback latency, reducing buffering and stutter.
- Refined the preview startup path so opening preview is less likely to block main-window loading and interaction.
- Continued the embedded playback direction so preview no longer depends on scattered external `ffplay` windows in Task Manager.

### Automatic Updates And Release Channels

- Integrated Velopack automatic updates, with the about panel showing the current build channel and offering manual update checks.
- Added stable and Beta update channels: stable uses `win`, and Beta uses `win-beta`.
- Beta builds or Beta channel selection now force session logging and automatic updates, keeping test users on the latest build and preserving diagnostics.
- Switching update channels now shows a confirmation dialog, saves the selected channel, and immediately checks the target channel. Switching from Beta back to stable can install the stable build automatically to complete the channel change.
- Unified application version, package version, and build-channel metadata. Stable builds show `v1.7.5`, while Beta builds show a Beta-marked package version.

### Settings, Logging And Developer Entry

- Reordered settings cards by usage flow: language, theme, update channel, session logs, tray, desktop shortcut, config file, and advanced tools.
- The session-log card now reflects whether logging can be disabled; forced Beta logging disables the toggle and explains why.
- Added persisted developer-mode state and moved developer tools to a lower-noise entry point for normal users.
- The about panel now shows the build channel so users can confirm whether the installed package is stable or Beta.

### Build, Installer And GitHub Release

- The build workflow now supports stable and Beta release inputs and passes consistent version/channel values through build, test, publish, Velopack packaging, and upload steps.
- Stable releases default to `v1.7.5`; Beta releases default to `v1.7.5-beta`, while Beta package versions use `1.7.5-beta.{run_number}` for continuous upgrades.
- Beta releases no longer require handwritten `docs/releases/*.md` notes; the workflow generates concise notes and commits automatically.
- Before publishing Beta, the workflow removes old Beta releases/tags and recreates the current prerelease so GitHub normally keeps a single Beta release slot.
- Reorganized release assets to keep the custom installer EXE, Portable ZIP, Velopack full package, and update manifests for installation, portable use, and in-app updates.
