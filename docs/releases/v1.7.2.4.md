## 中文

- 自 `v1.6.7.1` 以来，整理并合并 `7hz_dev` 分支的录制、解析、预览与便携发布更新。
- 优化设置页的定时、周期和视频分段输入，修复单位输入裁切问题，并补充保存命名与路径层级选项。
- 调整主界面快捷操作、房间标签、列表胶囊样式和多选状态，减少过度圆角并统一按钮间距。
- 修复视频列表导入、删除、移动和归类时的分段元数据保留问题。
- 修复直播流解析器的错误隔离、回退逻辑和输入校验，避免一个地址的失败污染其他任务。
- 强化录制稳定性、窗口响应式布局和便携运行时文件组织。
- 更新 README 与项目结构说明，明确 MPLR 当前仅支持 Windows。
- 统一版本号到 `Directory.Build.props`，移除旧的 macOS 打包残留版本源。
- 新增 GitHub Actions 自动构建、打包、上传构建产物和按 tag/manual 发布 GitHub Release 的流程。
- 修复 GitHub Actions Windows runner 上 `Split` collection expression 可能触发的编译兼容问题。
- 修复 GitHub Actions Windows runner 上 `Reverse()` 重载解析导致的 WPF 临时项目编译失败。

## English

- Consolidated recording, resolver, preview, and portable release updates from the `7hz_dev` branch since `v1.6.7.1`.
- Improved settings inputs for schedules, cycle periods, and video segment durations, including clipped unit fields and save naming/path options.
- Refined quick actions, room chips, list capsule styling, multi-select state handling, button spacing, and excessive corner radii.
- Fixed segment metadata preservation during video list import, delete, move, and categorization operations.
- Isolated resolver errors by URL and hardened fallback behavior and input validation.
- Improved recording stability, responsive window behavior, and portable runtime layout.
- Updated README and project structure notes to make Windows-only support explicit.
- Centralized versioning in `Directory.Build.props` and removed the stale macOS package version source.
- Added GitHub Actions automation for Windows build, packaging, artifact upload, and tag/manual GitHub Release publishing.
- Fixed a GitHub Actions Windows runner compile compatibility issue caused by `Split` collection expressions.
- Fixed a WPF temporary project compile failure caused by `Reverse()` overload resolution on the GitHub Actions Windows runner.
