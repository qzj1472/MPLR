### 中文

### 更新范围

- 本日志汇总自 `v1.6.8.1` 之后至 `v1.7.4.3` 的累计更新，整合 `v1.7.2.4`、`v1.7.3.1`、`v1.7.4.2` 与 `v1.7.4.3` 相关历史提交和发布说明。

### 录制与监控

- 强化直播录制稳定性，优化录制初始化、停止状态同步、子进程清理与异常恢复，降低停止后被重新拉起或重复录制的风险。
- 录制过程中解析短暂失败时会保留正在使用的可用直播流地址，避免临时网络或平台解析异常直接打断录制状态。
- 新增低电量保护与待转码队列，完善定时关机、停录、分段与转码相关流程，减少录制结束后的状态残留。
- 补齐双音轨录制、分段时长单位、分段大小单位和局部录制参数，录制格式、清晰度、分段、轮询周期、保存目录和命名规则可按直播间生效。
- 录制完成后自动清理没有对应视频文件的孤立元数据，减少无效 `.mplr.json` 文件残留。
- 增加运行资源日志，记录软件、`ffmpeg`、`ffplay`、`ffprobe` 等关键运行依赖状态，便于排查录制、预览和转码问题。

### 预览与解析

- 优化预览启动链路，后台预热解析器并缓存 Python、解析脚本、`ffplay`、`ffprobe` 路径，减少重复进程探测和首次等待时间。
- 预览优先使用直播流而不是本地录制文件，修复点击预览时可能从本地文件开头播放、无法看到实时画面的问题。
- 多个候选流会并发快速探测，已有可用流地址时优先直接打开，降低预览前的阻塞和等待。
- 修复预览窗口刚打开后手动拖动会自动复位的问题，改善预览窗口的交互稳定性。
- 隔离外部直播流解析器的临时配置，避免并发解析时配置互相覆盖，并自动清理过期临时配置。
- 加强抖音未开播房间头像解析与本地头像缓存，提升未开播或头像缺失时的房间信息展示质量。
- 加强直播流解析错误隔离、回退逻辑和输入校验，避免单个地址失败影响其他任务。

### 视频列表与媒体处理

- 扩展录屏列表能力，新增视频分割、连续片段合并、多选操作、合并条件校验和风险提示。
- 合并视频前会校验片段格式，阻止不同格式片段直接使用 `-c copy` 合并，降低输出异常或合并失败风险。
- 修复 FFmpeg concat 合并列表写入 UTF-8 BOM 后无法识别首行的问题，合并成功后会为输出视频保留元数据。
- 修复视频导入、删除、移动和归类时分段元数据丢失的问题。
- 限制缩略图生成与媒体探测并发，避免视频较多时同时拉起过多 FFmpeg 或 FFprobe 进程。
- 优化视频列表布局，多选操作、取消多选、删除图标、封面缓存和胶囊样式更统一。

### 设置、界面与交互

- 新增并完善直播间局部设置入口，可为单个直播间配置监控、录制、通知、录制格式、清晰度、分段、轮询周期、轮询时段、保存目录和命名规则。
- 修复局部设置错误写入全局配置的问题，直播间局部配置会随房间保存并在解析和录制时生效。
- 优化添加直播间流程，可在添加时直接选择是否跟随全局设置，以及是否启用监控和录制。
- 优化设置页的定时、周期、视频分段、保存命名和路径层级输入，修复单位输入裁切问题。
- 调整主界面快捷操作、房间标签、按钮间距、多选状态和过度圆角，统一按钮悬停、按下和运行状态样式。
- 优化窗口尺寸、弹窗定位、浮层居中和响应式布局，在多屏和不同 Windows 版本下显示更稳定。
- 隐藏加载窗口的任务栏入口，减少任务栏分组干扰。
- 新增通知合并与自定义汇总间隔，并恢复 Toast 覆盖式显示，减少高频通知干扰。
- 优化日志导出弹窗、默认日志策略、日志目录操作和配置变更日志读取，降低配置文件短暂占用导致的读取失败。

### 项目结构、构建与发布

- 整理项目结构和 README，明确当前仅支持 Windows。
- 统一版本号来源到 `Directory.Build.props`，减少多处手动维护版本的风险。
- 新增 GitHub Actions 自动构建、Windows 打包、构建产物上传和 tag/manual GitHub Release 发布流程。
- 修复 GitHub Actions Windows runner 上 `Split` collection expression 与 `Reverse()` 重载解析导致的编译兼容问题。
- 优化自动生成 Release Notes 的正文格式，移除重复版本信息、重复贡献者段落和多余分割线。
- 清理未使用资源、旧平台入口和过期 README 引用，完善便携运行时资源组织。

### English

### Scope

- This changelog summarizes cumulative updates from `v1.6.8.1` through `v1.7.4.3`, consolidating the historical changes covered by `v1.7.2.4`, `v1.7.3.1`, `v1.7.4.2`, and `v1.7.4.3`.

### Recording And Monitoring

- Improved live recording stability across initialization, stop-state synchronization, child process cleanup, and failure recovery, reducing the risk of stopped rooms being restarted or recorded twice.
- Preserved usable live stream URLs while recording when a later resolver check temporarily fails, preventing transient resolver or network issues from interrupting active recordings.
- Added low-battery protection and a pending transcode queue, and hardened auto-shutdown, stop-recording, segmentation, and transcoding flows.
- Completed dual-audio recording, segment duration units, segment size units, and per-room recording options. Format, quality, segmentation, polling interval, save folder, and filename rules can now apply per room.
- Automatically removes orphan metadata when no matching video file is produced after recording.
- Added runtime resource logging for the application and key tools such as `ffmpeg`, `ffplay`, and `ffprobe` to simplify recording, preview, and transcoding diagnostics.

### Preview And Resolving

- Sped up preview startup by warming the resolver in the background and caching Python, resolver script, `ffplay`, and `ffprobe` paths.
- Restored live-stream-first preview behavior instead of replaying local recording files from the beginning.
- Probes multiple candidate streams in parallel and opens cached playable URLs directly when available, reducing preview wait time.
- Fixed preview windows snapping back after being manually moved right after opening.
- Isolated temporary resolver configuration files so concurrent resolving tasks no longer overwrite each other, with stale temp configs cleaned up automatically.
- Improved Douyin offline avatar resolution and local avatar caching for better room information display when a room is offline or missing an avatar.
- Hardened resolver error isolation, fallback behavior, and input validation so one failed URL is less likely to affect other tasks.

### Video List And Media Processing

- Expanded the screen-record list with video splitting, continuous segment merging, multi-select actions, merge validation, and warning prompts.
- Validates segment formats before merging to avoid unsafe `-c copy` concatenation across incompatible files.
- Fixed FFmpeg concat failures caused by UTF-8 BOM in merge list files, and preserves metadata for successfully merged output videos.
- Fixed segment metadata preservation during video import, delete, move, and categorization.
- Throttled thumbnail generation and media probing to avoid spawning too many FFmpeg or FFprobe processes at once.
- Refined video-list layout, multi-select actions, cancel selection, delete icon, thumbnail cache, and capsule styling.

### Settings, UI And Interaction

- Added and completed per-room local settings for monitor, recording, notification, recording format, stream quality, segmentation, polling interval, polling schedule, save folder, and filename rules.
- Fixed local room settings being written to global configuration; per-room settings are now saved with the room and applied during resolving and recording.
- Improved the add-room flow so global-setting inheritance, monitoring, and recording can be selected while adding a room.
- Improved settings inputs for schedules, cycle periods, video segments, save naming, and path levels, including fixes for clipped unit fields.
- Refined quick actions, room chips, button spacing, multi-select states, excessive corner radii, and top run-button hover and pressed states.
- Improved window sizing, dialog positioning, floating-panel centering, and responsive layout across multi-display setups and different Windows versions.
- Hid loading windows from taskbar grouping to reduce taskbar clutter.
- Added merged notifications with a configurable summary interval and restored overlay-style toast display.
- Improved log export dialogs, default logging behavior, log folder actions, and config-change log reading to reduce failures caused by short-lived config file locks.

### Project Structure, Build And Release

- Refined the project structure and README, making Windows-only support explicit.
- Centralized versioning in `Directory.Build.props` to reduce multi-file version maintenance.
- Added GitHub Actions automation for Windows build, packaging, artifact upload, and tag/manual GitHub Release publishing.
- Fixed GitHub Actions Windows runner compatibility issues caused by `Split` collection expressions and `Reverse()` overload resolution.
- Refined generated Release Notes formatting by removing repeated version information, duplicate contributors content, and extra separators.
- Cleaned unused assets, legacy platform entries, stale README references, and improved portable runtime resource organization.
